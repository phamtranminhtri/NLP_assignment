\section{Literature review}

Before we dive into the problem, let's review the 2 important paper that explain how the 2 models we will be using work: BERT and SBERT.

\subsection{BERT}

BERT (Bidirectional Encoder Representations from Transformers) was introduced in the paper ``BERT: Pre-training of Deep Bidirectional Transformers for
Language Understanding'' by Devlin et al at Google AI Language in 2019. Unlike other language models, like the original ``Attention is all you need'' paper (encoder-decoder) or GPT (decoder only), BERT has encoder only architecture, and it was pretrained without future masking, allowing information to flow in 2 direction, left to right and right to left, making the model able to represent better the whole sequence. The pretrained model is then finetuned for various downstream tasks.

There are 2 step in BERT framework: pre-training and fine-tuning. The model is first pretrained on unlabeled data. The fine-tuning step take the pre-trained BERT model and train the parameters with their respective labeled data.

BERT's architecture is a multiple transformer encoder layer, with 2 model sizes, BASE (768d hidden size, 110M params) and LARGE (1024d hidden size, 340M params). BERT allow the input to be either a single sentence or a pair of sentence (question-answer, next sentence, etc.). It use WordPiece embedding with vocab size of 30000. The first token is always \texttt{CLS} for classification tasks. If the input is a pair of sentences, there will be a \texttt{SEP} token between them.

BERT was pre-trained on BooksCorpus (800M words) and English Wikipedia (2500M words), with 2 tasks: Masked Language Modeling (MLM) and Next Sentence Prediction (NSP).

\begin{itemize}
    \item In MLM, random token are masked and the model has to predict them. Because we are predicting the \texttt{MASK} token instead of the next token (like GPT), we can allow  attention to flow to both past and future token, allowing bidirectional information
    \item In NSP, we choose a pair of sentences and have the model predict whether sentence B is the next sentence of sentence A in the corpus. This task help the model understand relationship between sentences and improve downstream task, like Question Answering (QA) and Natural Language Inference (NLI)
\end{itemize}

For fine-tuning, BERT was trained end-to-end from the pre-trained parameters with the tasks' input and output. At the time it achieved SOTA on 11 tasks:

\subsection{SBERT}

BERT is a powerful model for multiple downstream task. But  when it come to semantic textual similarity, specifically finding similarity between 2 sentences, it require feeding both sentences into the network pairwise. Which mean, given $n$ sentences, we need to calculate $\frac{n(n-1)}{2}$ combination of sentence pair to get the similarity between each pair, which is inefficient. Another method is to get embedding of each sentence individually, to compute the similarity between the embedding vectors. It's common to use the last hidden state to represent the sentence, such as the \texttt{CLS} token, or the average pool of the embeddings.
SBERT paper argues that this approad yeild worse embedding result than other existing approachs, like GLoVe.

SBERT (Sentence-BERT) was introduced in the paper ``Sentence-BERT: Sentence Embeddings using Siamese BERT-Networks'' by Nils Reimers et al in 2019. SBERT addressed the mentioned issues by taking BERT and training it on their specific task suitable for encoding the sentence for sementic similarity and clustering tasks.

\begin{figure}[!h]
    \centering
    \includegraphics[width=0.3\linewidth]{image/sbert_eval.png}
    \caption{SBERT inference}
    \label{fig:sbert_eval}
\end{figure}

SBERT add a pooling layer to the output of BERT, with many availble config (\texttt{CLS}, \texttt{MEAN}, \texttt{MAX}), thought the default is \texttt{MEAN}, and use cosine similarity as the default similarity function.

SBERT used the following objective function:

\begin{itemize}
    \item \textbf{Classification:} With 2 sentence embedding $u$ and $v$:
          \[
              o = \mathrm{softmax}\!\left(W_t(\text{concat}(u, v, |u - v|))\right)
          \]
          we then use cross entropy loss
    \item \textbf{Regression:} Compute the cosine similarity between 2 embedding, then use mean square error loss
    \item \textbf{Triplet:} Given an anchor
          sentence $a$, a positive sentence $p$, and a negative
          sentence $n$, we pull $p$ closer to and $n$ further away from $a$.
          \[
          \max\left(\lVert s_a - s_p \rVert - \lVert s_a - s_n \rVert + \epsilon,\; 0\right)
          \]

\end{itemize}


SBERT was trained on SNLI and  Multi-Genre NLI datasets with Adam optimizer and linear learning rate warm-up. It achived remarkable results on multiple benchmark. The best selling point for SBERT is its efficency in encoding sentences with same trade off for accuracy.
